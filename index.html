Core Status Definitions (狀態定義)

  • Submitted

    New ticket. Includes SLA tracking for response time.
    新工單建立，系統自動追蹤 SLA 回應時效。

  • CS OK

    Gatekeeper check. Ensures "Definition of Ready" is met (Logs/Screenshots).
    守門員檢查，確保資訊（Log/截圖）符合 Ready 定義。

  • Wait for Info

    (Loop State) Holding for customer/internal logs.
    (循環狀態) 等待補充資訊，暫停 SLA 計算。

  • Tech Assigned

    Confirmed Bug. Handoff to Engineering backlog.
    確認為 Bug，正式移交工程團隊 Backlog。

  • Sol. Proposed

    Engineering has implemented code changes and released the fix.
    工程團隊已完成代碼變更並釋出修復方案。

1 The Process of Solving Issues (解決問題的邏輯)

Standardized SOP for troubleshooting (標準化除錯流程):

  1. 1. Collect Info: Gather Logs, IDs, timestamps. (蒐集 Log、ID、時間戳)
  2. 2. Clarify Expectations: Expected vs. Actual behavior. (釐清預期行為 vs. 實際行為)
  3. 3. Reproduce: Replicate in Staging environment. (在測試環境重現問題)
  4. 4. Root Cause Analysis: Identify source (Code/Data/User). (根本原因分析：代碼/數據/使用者？)
  5. 5. Solve/Delegate: Fix it myself or assign to Eng. (自行修復或指派給工程師)
  6. 6. Document: Update Knowledge Base. (更新知識庫，避免重複發生)

2 Standard Issue Template

Source: Customer / Internal // 來源

Description: Brief summary of the issue. // 簡述

Expected Behavior: What should happen? // 預期行為

Actual Behavior: What is happening? // 實際行為

Steps to Reproduce: 1. Login -> 2. Click... // 重現步驟

Environment: Production / Staging, Browser // 環境

Attachments: Screenshots, Logs, HAR files // 附件
